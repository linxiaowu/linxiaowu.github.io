Lambda 表达式，本质上就是可以传递给其他函数的一小段代码。这章主要讲了：

Lambda 表达式和成员引用

在代码中存储和传递一小段行为是常有的任务。例如

  1. 当一个事件发送的时候运行这个事件处理器。
  2. 把这个操作引用到这个数据结构中所有的元素上。

在 Java 中，可以用匿名内部类来实现。但这种做法太啰说。

函数式编程提供了另一种解决问题的方法：把函数作为值来对待。可以直接传递函数。而不需要先声明一个类再传递这个类的实例。使用 lambda 表达式会更加简洁。

和 Java 不一样，Kotlin 允许在 lambda 内部访问非 final 变量甚至修改它们。从 lambda 内部访问外部变量。我们称这些变量被 lambda捕捉。

惰性集合操作:序列

在链式集合函数调用的例子，例如 map 和 filter。这些函数都会及早地创建中间集合，也就是说每一步的中间结果都被存储在一个列表。序列给了你执行这些操作的另一种选择。可以避免创建这些临时中间对象。

从上面的例子我们知道，filter 和 map都会返回一个列表。这意味着上面例子中的链式调用会创建两个列表。如果元素少，那没什么问题，但是元素一多。调用就会变得十分低效了。

为了提高效率，可以把操作变成使用序列，而不是直接使用集合：

Kotlin 惰性集合操作的入口就是 Sequence 接口。这个接口表示的就是一个可以列举元素序列。Sequence 只提供一个方法，iterator，用来从序列中获取值。

执行序列操作：中间和末端操作

序列操作分为两类：中间的和末端的。一起中间操作返回的是另一个序列，这个新序列知道如何变换原始序列中的元素。而一次末端操作返回的是一个结果，这个结果可能是集合、元素、数字。

像 map 和 filter 这些属于中间操作。而 toList 属于末端操作。

map(1) filter(1) map(2) filter(4) map(3) filter(9) map(4) filter(16)

可以看出。它的执行顺序是，按照应用在每一个元素上 : 处理完第一个元素（先映射再过滤），然后再完成第二个，以此类推。

对于序列操作来说，意味着你可以跳过处理部分元素。

当然，在集合中执行操作顺序也会影响性能。例如先 filter 再map ，性能优于先 map 再 filter。

总结

  • Lambda 允许把代码块当做参数传递给函数
  • 可以把 Lambda 放在括号外传递给函数，而且可以用 it 引用单个 lambda 参数
  • Lambda 中的代码可以访问和修改包含这个 lambda 调用的函数中的变量
  • 通过在函数名称前加上前缀 :: ，可以创建方法、构造方法及属性的引用，并用这些引用替代 lambda 传递给函数
  • 使用 filter map all any 等函数，大多数公共的集合操作都不需要手动迭代元素就可以完成
  • 序列允许你合并一个集合的多次操作，而不需要创建新的集合来保存中间结果
  • 可以把 lambda 作为实参传给接收 Java 函数式接口（带抽象方法的接口，也可以叫做 SAM 接口）作为形参的方法。
  • 带接受者的 lambda 是一种特殊的 lambda，可以在这种 lambda 中直接访问一个特殊接受者对象的方法。
  • with 标准库函数允许你调用同一个对象的多个方法，而不需要反复的写出这个对象的引用。 apply 函数让你使用构建者的风格的 API 创建和初始化任何对象
